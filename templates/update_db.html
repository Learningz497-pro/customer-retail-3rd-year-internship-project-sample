File Requirements

Ensure your CSV file contains exactly these headers:

  • CustomerID: Numeric ID (e.g., 12345)
  • Recency: Days since last purchase
  • Frequency: Number of orders
  • Monetary: Total spend in ₹
  • Cluster: K-Means cluster ID (e.g., 0, 1, 2)
  • Segment: "VIP", "Regular", or "At Risk"
SQLite Integration

When you upload a valid dataset, the following actions occur automatically:

  1. The system parses and validates the CSV structure.
  2. The local customer_segments.csv file is completely replaced with the new data.
  3. A secure connection is made to the local database.db SQLite instance.
  4. The table is overwritten, syncing your historical record.
  5. The global memory state is refreshed, updating the Dashboard instantly.
